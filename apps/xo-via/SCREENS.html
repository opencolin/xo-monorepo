XO Phone

Screens the user sees

Every screen in v.Next as a phone-frame mockup, with what's on it, what the user can do, where they came from, and what data sources back it.

companion to V_NEXT_PLAN.html section-by-section fill-in last updated: 2026-05-20

What's new in this revision. Chat is the leftmost dock button (no separate widget). Three overlays consolidated into one (top swipe-down anywhere). Agent mode has iOS-style sub-pages (cowork apps push it past one screen). Agent can drive the OS via client-action SSE events. Sign-in is required, gated after the Via demo. Dock middles stay empty for v.Next.

What's happening

  • Chat is expanded; user asked "show me my files"
  • Assistant text streaming: "Sure, opening your files..."
  • SSE event arrives: event: client-action with {kind:"navigate", route:"/files"}
  • lib/agent/actions.ts dispatcher reads the action, calls PhoneContext.openApp("/files") + router.push
  • OS routes to /files (Files cowork app) in the background
  • Toast shows "Opening Files..." for ~1s
  • When user taps back chevron: chat collapses, /files (now loaded) is visible underneath

Action kinds supported (Section J)

  • {kind:"navigate", route:"/files"} → openApp + router.push
  • {kind:"go-home"} → goHome
  • {kind:"pop-back"} → pop back stack
  • {kind:"open-overlay", section?} → opens consolidated Overlay
  • {kind:"close-overlay"}
  • {kind:"collapse-chat"} → closes chat surface
  • {kind:"expand-chat"} → opens chat surface
  • Unknown kinds ignored (forward compatible)

Data sources

  • cowork-api chat stream extended with client-action event type (backend work)
  • Phone OS subscribes to same EventSource it uses for text-delta
  • Action dispatcher in lib/agent/actions.ts maps kind → OS-level call

Pending

  • Action timing: route immediately or wait for assistant text turn to finish?
  • Background sessions (stream stays open when chat collapses)?
  • Toast / pill UX while action fires
