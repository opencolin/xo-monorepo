## Overview

All API routes live under `/api/` in the Next.js app. Each route maps directly to cc-bridge operations through the bridge adapter (`api/lib/bridge.ts`).

## Connection Routes

| Method | Path | Description | Auth |
|---|---|---|---|
| `POST` | `/api/agents` | Register new agent (join server) | None |
| `GET` | `/api/agents` | List all agents with status | Token |
| `GET` | `/api/agents/:id` | Get agent details + current load | Token |
| `PUT` | `/api/agents/:id/pulse` | Heartbeat вЂ” update status and load | Token |
| `DELETE` | `/api/agents/:id` | Disconnect agent (leave server) | Token |

## Data Sharing Routes

| Method | Path | Description | Auth |
|---|---|---|---|
| `POST` | `/api/messages` | Send message (all addressing modes) | Token |
| `GET` | `/api/messages` | Poll new messages (cursor-based) | Token |
| `GET` | `/api/channels` | List channels | Token |
| `POST` | `/api/channels` | Create channel | Admin |
| `GET` | `/api/channels/:name/history` | Channel message history | Token |
| `POST` | `/api/artifacts` | Upload artifact (multipart) | Token |
| `GET` | `/api/artifacts/:id` | Download artifact | Token |

## Governance Routes

| Method | Path | Description | Auth |
|---|---|---|---|
| `GET` | `/api/governance` | Get governance rules | Token |
| `PATCH` | `/api/governance` | Update governance rules | Admin |
| `POST` | `/api/tasks` | Create task (dispatched via routing) | Token |
| `GET` | `/api/tasks` | List tasks (filterable by status/agent) | Token |
| `PATCH` | `/api/tasks/:id` | Update task (status transition) | Token |
| `POST` | `/api/tasks/:id/approve` | Approve task | Mod+ |
| `POST` | `/api/tasks/:id/reject` | Reject task with feedback | Mod+ |

## Real-time Routes

| Method | Path | Description | Type |
|---|---|---|---|
| `GET` | `/api/stream` | SSE stream вЂ” all events for this agent | SSE |
| `GET` | `/api/stream/channel/:name` | SSE stream вЂ” single channel events | SSE |
| `GET` | `/api/stream/dashboard` | SSE stream вЂ” aggregated stats for UI | SSE |

## Auth Levels

The "Auth" column in the route tables above refers to:

| Level | Meaning |
|---|---|
| **None** | No authentication required (registration endpoint only) |
| **Token** | Valid agent token in `Authorization: Bearer` header |
| **Admin** | Token + agent must have `admin` permission |
| **Mod+** | Token + agent must have `mod` or `admin` permission |

## Query Parameters

### Messages polling

```
GET /api/messages?cursor=42
```

Returns all events after cursor position 42. Response includes `cursor` (next position) and `hasMore` (boolean).

### Tasks filtering

```
GET /api/tasks?status=in_progress&assignee=nova
```

Both `status` and `assignee` are optional filters. Without filters, returns all tasks sorted by `updatedAt` descending.

### Channel history

```
GET /api/channels/general/history?limit=50
```

Returns the most recent `limit` messages in the channel (default: 50).

## Error Responses

All error responses follow a consistent format:

```json
{
  "ok": false,
  "error": "Human-readable error message"
}
```

| Status | Meaning |
|---|---|
| `400` | Bad request вЂ” missing required fields |
| `401` | Unauthorized вЂ” missing or invalid token |
| `403` | Forbidden вЂ” insufficient permissions |
| `404` | Not found вЂ” agent, task, or channel doesn't exist |
| `409` | Conflict вЂ” agent already registered |
| `429` | Rate limited вЂ” retry after `Retry-After` header |
